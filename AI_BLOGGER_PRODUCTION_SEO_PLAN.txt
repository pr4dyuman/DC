PURPOSE
This document captures:
- what the current AI Blogger workflow in DC actually does
- what the ideal production SEO workflow should do
- the phased implementation plan to turn AI Blogger into a high-quality SEO content system

CORE SEO PRINCIPLES
- Do not chase trends just because they are trending.
- Choose topics that match the business, the offer, and the buyer intent.
- Build people-first content that is useful, specific, and trustworthy.
- Use clear metadata, structured internal linking, and a review gate before publishing.
- Treat AI as a drafting and acceleration layer, not as the only source of truth.

4. WHAT THE IDEAL PRODUCTION SEO WORKFLOW SHOULD BE
======================================================================

STEP 1. TOPIC SOURCE COLLECTION
Inputs can come from:
- live trends
- website crawl
- keyword cluster
- manual campaign brief
- content calendar / schedule

STEP 2. BUSINESS RELEVANCE FILTER
Before a topic is accepted, answer:
- Does this fit the company services or products?
- Does this attract the right audience?
- Is the intent useful for this business?
- Can this topic naturally lead to a CTA?

STEP 3. CANNIBALIZATION CHECK
Before drafting:
- search existing posts
- compare keyword overlap
- check whether a parent cluster page already exists
- avoid publishing another post that targets the same keyword with the same intent

STEP 4. SERP INTENT AND COMPETITOR ANALYSIS
For the chosen topic:
- fetch real SERP results
- classify search intent
- detect content format patterns
- extract common headings
- identify missing angles
- estimate realistic ranking difficulty

STEP 5. WEBSITE INTELLIGENCE
If a website is provided:
- crawl service pages
- crawl top blog pages
- extract tone, offer, positioning, common themes
- build internal link targets
- build banned claims and preferred language signals

STEP 6. RESEARCH
Gather:
- source-backed facts
- examples
- objections
- supporting data
- FAQs
- practical steps

STEP 7. SEO BRIEF CREATION
Build a proper brief with:
- selected topic
- primary keyword
- secondary keywords
- entities
- search intent
- target audience
- internal links to include
- CTA goal
- FAQ suggestions
- title direction
- metadata direction

STEP 8. DRAFT GENERATION
Generate:
- title
- meta title
- meta description
- outline
- content
- FAQ section if needed
- tags
- suggested image concept
- alt text

STEP 9. HARD SEO QA
Do not rely only on AI opinion.
Run rule-based validation:
- primary keyword exists
- title exists and is strong
- meta description exists
- H2 structure exists
- internal links exist
- content is within range
- CTA exists
- excerpt exists
- canonical/meta fields exist
- schema eligibility check passes

STEP 10. EDITORIAL REVIEW
Human or admin review should verify:
- factual safety
- brand tone
- uniqueness
- sales alignment
- legal / claims risk
- publish timing

STEP 11. PUBLISHING
Publish package should include:
- title
- slug
- excerpt
- meta title
- meta description
- canonical URL
- image
- alt text
- structured data
- category
- tags

STEP 12. POST-PUBLISH MONITORING
After publish:
- track impressions
- track CTR
- track rankings
- detect underperformers
- refresh content after defined interval

Output:

- Public blog pages become metadata-ready.

Priority:
- Highest

PHASE 2. HARD SEO VALIDATOR
Goal:
Turn SEO checks from "suggestions" into hard validation and scoring.

Tasks:
- Build a rule-based SEO audit helper.
- Score:
  - keyword coverage
  - excerpt / meta description presence
  - internal link count
  - heading structure
  - word range
  - CTA presence
  - FAQ presence when needed
  - image alt text
- Add blocking rules before moving to Approved or Scheduled.
- Show failed checks clearly in post detail and editor pages.

Output:
- Approval is quality-gated.
- SEO score becomes measurable and not just AI opinion.

Priority:
- Highest

PHASE 3. REAL WEBSITE INTELLIGENCE
Goal:
Make Website mode real.

Tasks:
- Add a website crawl provider setting in AI Blogger admin.
- Build a crawler step that:
  - fetches homepage
  - fetches top navigation/service pages
  - extracts headings, offers, FAQs, tone hints
  - extracts existing internal links
  - identifies blog categories and service priorities
- Save crawl summaries in MongoDB for reuse.
- Feed crawl summary into topic selection and drafting.

Output:
- Website mode becomes a real business-aware input path.

Priority:
- Highest

PHASE 4. SERP AND COMPETITOR ANALYSIS
Goal:
Understand ranking reality before drafting.

Tasks:
- Add SERP provider setting in AI Blogger admin.
- Build a SERP analysis step for selected topic / keyword.
- Capture:
  - top result titles
  - heading patterns
  - likely intent
  - weak spots / missing angles
  - featured snippet style
  - People Also Ask style questions if available
- Save SERP snapshots in MongoDB.

Output:
- Drafts are based on search reality, not just AI guessing.

Priority:
- Highest

PHASE 5. INTERNAL LINK ENGINE
Goal:
Make internal linking strategic instead of manual guesswork.

Tasks:
- Build an internal-link map from:
  - service pages
  - existing blog posts
  - key landing pages
- For every new post, suggest:
  - 2 to 5 internal target pages
  - recommended anchor text
  - cluster parent page
- Add approval rule:
  - post must include minimum internal links before approval if target is DC blog

Output:
- Better crawl flow
- better topical authority
- stronger conversion path

Priority:
- High

PHASE 6. GROUNDED RESEARCH LAYER
Goal:
Upgrade research from AI-only to source-backed.

Tasks:
- Add source retrieval step.
- Save source list with:
  - title
  - URL
  - summary
  - type
  - freshness
  - trust level
- Feed only selected sources into the draft prompt.
- Keep citations or source notes in the draft metadata for editor review.

Output:
- Research becomes more trustworthy and useful.

Priority:
- High

Produce better publish-ready output.

  - service explainer

Priority:
- High

Make the public page technically stronger.

Tasks:

Make schedules actually execute.

Output:

Priority:
- Medium

PHASE 11. IMAGE SYSTEM

Stop publishing with a placeholder image path.
