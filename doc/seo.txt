1. Crawlability and indexability
- Public pages must return HTTP 200.
- Removed pages must return 404 or 410, not soft-404 content.
- Redirect old URLs with 301 redirects to the best current URL.
- Keep one preferred host only: https://digitalcorvids.com.
- Redirect www to non-www, already present in next.config.ts.
- Do not allow staging or preview domains to be indexed.
- Do not block CSS, JS, images, or fonts that Google needs to render public pages.
- Private pages must use noindex metadata and authentication.
- Dashboard, super-admin, login, admin, API, trial-expired, and plan-expired pages should not be indexed.
- Use X-Robots-Tag for non-HTML files only when needed.
- Do not rely on robots.txt to hide sensitive pages. Use noindex, authentication, password protection, or remove the page.

2. Robots.txt
- Keep public/robots.txt available at /robots.txt.
- Keep Sitemap: https://digitalcorvids.com/sitemap.xml.
- Disallow crawl-heavy private areas:
  - /api/
  - /dashboard
  - /super-admin
  - /admin
- Allow public marketing pages, blogs, images, CSS, and JS.
- Add AI crawler policy only after deciding the business intent:
  - Allow if the goal is broader AI discovery.
  - Disallow if the goal is restricting training/crawling.
- Candidate AI crawler user agents to review:
  - GPTBot
  - ChatGPT-User
  - ClaudeBot
  - PerplexityBot
  - Google-Extended
  - CCBot
  - Bytespider
- Remember: robots.txt controls crawling access, not guaranteed indexing removal.

4. Canonical URLs and duplicates
- Every public page should have one canonical URL.
- Canonicals must be absolute, for example https://digitalcorvids.com/services/seo.
- Canonical, sitemap URL, Open Graph URL, and internal links should all match.
- Normalize trailing slashes consistently.
- Avoid duplicate content across service pages.
- Redirect legacy .html pages to current routes, already partly handled in next.config.ts.
- Make sure blog canonical URLs do not accidentally point to another domain unless intentionally syndicated.
- Avoid query parameter duplicates for indexable pages.
- Add canonical handling for paginated blog/category pages if pagination is added.

5. Rendering and JavaScript SEO
- Keep important text in server-rendered HTML.
- Do not hide primary content behind client-only state, canvas, animations, or API calls.
- The home page currently uses client-side rendering; convert it to a Server Component and isolate Swiper/animations into small client components.
- Navigation links must be real links using next/link or a href.
- Each meaningful page or screen should have a unique URL.
- Avoid injecting title, description, canonical, and robots metadata only after hydration.
- Use server-rendered metadata through Next.js Metadata API.
- Lazy-load below-fold interactive sections.
- Avoid dynamic rendering as a long-term SEO workaround.
