2B. FEATURES WITH PARTIAL/NO UI COVERAGE ⚠️

14. SERP Strategy Layer (Item 9 Backend Features)
    Backend: assessRankingDifficulty, detectDominantContentFormat,
             extractTitleAnglePatterns, buildSectionGapAnalysis
    UI: NOT surfaced anywhere in the dashboard
    IMPACT: Medium — these are deep analysis functions that run
            during generation but their outputs aren't stored as
            post fields or shown in any UI card
    STATUS: ⚠️ BACKEND ONLY (by design — consumed during AI generation)

15. Grounded Research Improvements (Item 10 Backend Features)
    Backend: extractKeyClaims, buildCitationBlock, source diversity
    UI: NOT surfaced — the source card shows type/trust/freshness
        but NOT key claims, citation blocks, or diversity scores
    IMPACT: Low — these are generation-time utilities
    STATUS: ⚠️ BACKEND ONLY (by design — consumed during AI generation)

16. Author / Entity Modeling (Item 11)
    Backend: AIBloggerAuthorConfig, AIBloggerEntityModelingConfig,
             buildBlogStudioArticleJsonLd
    UI: NOT visible in dashboard or super-admin UI. Author name,
        bio, social profiles, organization details, schema toggles
        are configured via super-admin API but have NO UI fields
    IMPACT: HIGH — super-admin needs UI fields to configure author
            and entity modeling settings
    STATUS: ⚠️ NO UI — CONFIG FIELDS NEED SUPER-ADMIN UI

17. Image System Maturity (Item 12)
    Backend: BlogStudioImageMeta (dimensions, format, size, license),
             BlogStudioImageCrop, BlogStudioImageHistoryEntry,
             imageHistory tracking in updatePost
    UI: Featured Image card shows only URL, alt text, and source.
        Does NOT show image dimensions, format, file size, crop
        coords, or replacement history
    IMPACT: Medium — image metadata would be useful in the editor
    STATUS: ⚠️ PARTIAL — image metadata and history not surfaced

18. Publish Package Validation (Item 13)
    Backend: validateBlogStudioPublishPackage with 8-category check
             (metadata, content, image, internal-links, schema,
             cannibalization, seo-score, approval)
    UI: Existing SEO Audit shows blockers/checks. The new detailed
        publish validation with fix hints is NOT called or shown.
        The publish status controls don't use this new validator
    IMPACT: HIGH — this is the whole point of Item 13, it should
            gate the publish action with clear blocker explanations
    STATUS: ⚠️ NOT INTEGRATED — validator exists but not wired into
            the publish flow or UI

==============================================================
SECTION 4: ENHANCEMENT RECOMMENDATIONS
==============================================================

PRIORITY 1 — CRITICAL (should do):

  E1. Wire Publish Validation into Publish Flow
      - Call validateBlogStudioPublishPackage before allowing
        status change to "Published"
      - Show blockers with fixHint as a modal or inline panel
      - Prevent publish if canPublish === false
      - Where: AIBloggerPostStatusControls component

  E2. Add Author/Entity Config UI in Super-Admin
      - Add form fields for author (name, URL, bio, social profiles)
      - Add organization fields (name, url, logo, services)
      - Add schema toggles (enableArticle, enableFaq, enableBreadcrumb)
      - Where: super-admin/ai-blogger/agency page or a new
        "Author & Entity" section in settings

PRIORITY 2 — RECOMMENDED (nice to have):

  E3. Show Image Metadata in Featured Image Card
      - Display image dimensions (width x height)
      - Show file format and size
      - Show license/attribution if present
      - Where: Post Detail → sidebar "Featured Image" card

  E4. Show Image Replacement History
      - List previous featured images with timestamps
      - Show who replaced it and why
      - Collapsible section under the Featured Image card
      - Where: Post Detail → sidebar "Featured Image" card

  E5. Surface Key Claims in Grounded Research Card
      - Show extracted claims per source (stats, quotes)
      - Display citation-ready blocks
      - Add source diversity score summary
      - Where: Post Detail → "Grounded Research" card

  E6. Surface SERP Strategy Outputs When Available
      - Show ranking difficulty score
      - Show dominant content format detection
      - Display title angle patterns
      - Show section gap analysis
      - Where: Post Detail → "SERP Snapshot" card (add a second
        section below the existing SERP data)

PRIORITY 3 — POLISH (optional):

  E7. Add Publish Validation Badge to Posts List
      - Show a small "Ready" / "X Blockers" badge on each post
        in the posts list view for quick scanning
      - Where: Post List page → each post row

  E8. Add Schema Markup Preview in Post Detail
      - Show the generated JSON-LD as formatted code block
      - Add "Copy Schema" button for manual use
      - Where: Post Detail → new "Schema Markup" card in sidebar

  E9. Show Readiness Score on Overview Recent Drafts
      - Add a "ready" / "needs work" indicator on each recent
        draft card on the overview page
      - Where: Overview → "Recent Drafts" section
