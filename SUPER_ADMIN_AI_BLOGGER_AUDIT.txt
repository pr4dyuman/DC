SECTION 2: OVERVIEW PAGE AUDIT
==============================================================

File: app/super-admin/ai-blogger/page.tsx (137 lines)

What It Shows:
  - Page title "AI Blogger Admin" with Brain icon + gradient badge
  - "Separate from system AI settings" label
  - Config-first context note
  - Agency list (all agencies, not just AI Blogger enabled ones)

Per Agency Row:
  ✅ Agency name (truncated, bold)
  ✅ Plan badge (uppercase)
  ✅ Status badge
  ✅ "AI Blogger on" / "AI Blogger off" badge (green/muted)
  ✅ User count, project count, client count
  ✅ Published posts count (from getBlogStudioOverviewImpl)
  ✅ Refresh candidates count (amber TrendingUp icon)
  ✅ Top refresh signal (post title + opportunity summary)
  ✅ "Manage AI" button → links to /agency/[id]

Behavior:
  - If an agency is already in context, redirects to /agency/[id]
  - Fetches overview data in parallel for all AI Blogger agencies

ISSUES:
  ⚠️ I1. No search/filter for agencies
     When there are many agencies, scrolling through all of them is
     cumbersome. Adding a search input would help.
     PRIORITY: LOW (most deployments have <20 agencies)

  ⚠️ I2. No readiness summary per agency
     The agency row shows published/refresh counts but doesn't show
     a quick at-a-glance readiness indicator (e.g. "5/6 ready" or
     "2 issues") from the readiness checks in the config page.
     PRIORITY: MEDIUM

  ⚠️ I3. No total/aggregate stats banner
     There's no platform-wide totals bar (e.g. "42 total published,
     7 refresh candidates across all agencies").
     PRIORITY: LOW

      Replace the centered spinner with section-level skeleton
      placeholders for a more polished loading experience.
